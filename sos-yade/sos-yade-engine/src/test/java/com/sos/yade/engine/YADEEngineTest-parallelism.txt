# Summary
# 
# Parallelism significantly improves performance for many small files 
# For large files, parallel execution often increases per-file duration 
#   Possible causes include:
#       Network bandwidth saturation
#       Server-side behavior (e.g. synchronization or throttling mechanisms)
#       Synchronization or contention within the underlying transfer library
#       ...
# 
# Protocol-dependent behavior is clearly visible (HTTP and WebDAV benefit most from parallelism)
